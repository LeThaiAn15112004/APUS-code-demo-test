Lab nhỏ: hiểu cách XSS xảy ra

XSS xảy ra khi dữ liệu do người dùng nhập bị đưa vào trang như HTML thật. Trình duyệt không còn xem đó là chữ nữa, mà có thể phân tích thành thẻ, thuộc tính sự kiện, hoặc JavaScript.

1. Nhập thử payload

Dán một trong các chuỗi này vào ô bên dưới:

<img src=x onerror="alert('XSS chạy rồi')">
<b style="color:red">Tôi đã chèn HTML vào trang</b>

2. Cách dễ bị tấn công: innerHTML

Vùng này dùng innerHTML, nên input có thể bị trình duyệt hiểu là HTML thật.

3. Cách an toàn hơn: textContent

Vùng này dùng textContent, nên input luôn được xem là chữ. Payload sẽ hiện ra nguyên văn và không chạy.

Điểm mấu chốt

Lỗi không nằm ở việc người dùng nhập ký tự lạ. Lỗi nằm ở chỗ ứng dụng tin dữ liệu đó và nhét thẳng vào DOM bằng API có thể diễn dịch HTML. 

unsafeOutput.innerHTML = input;   // nguy hiểm nếu input chưa được làm sạch
safeOutput.textContent = input;   // an toàn hơn khi chỉ cần hiển thị chữ
